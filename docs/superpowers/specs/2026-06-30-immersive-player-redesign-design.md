# MiningQwQMusic 沉浸式界面重构设计

## 目标

将现有三栏卡片式播放器重构为以音乐内容为中心的沉浸式主舞台，同时保留播放、账户、歌单、一起听、历史返回和现有 API 行为。

- 移动概念图：`C:\Users\ksy06\.codex\generated_images\019f1833-1e51-77c0-933b-7a36689b655d\call_7gXWWsUSZwfyOzAlRkZvBEyg.png`

界面使用黑曜石底色与由当前封面驱动的低饱和冷色环境光。玻璃材质仅用于悬浮导航、播放坞和临时抽屉。内容区依靠图像、留白和排版建立层级，不使用通用 Bento 仪表盘。

## 空间结构

- 桌面左侧使用无品牌 Logo 的悬浮图标胶囊，包含首页、搜索、音乐库、一起听、个人与设置入口。
- 主内容占据完整画布；常驻歌词和队列右栏删除。
- 首页由一张主专题和横向推荐轨道构成。
- 搜索页使用固定命令区与轻量结果列表。
- 音乐库使用分段导航、封面集合与紧凑列表。
- 桌面播放坞悬浮在底部中央；移动端使用迷你播放条与底部悬浮导航。
- 歌词保留在沉浸播放详情中；队列继续通过播放坞按钮打开抽屉。

## 动效

- 页面内容使用短距离错峰进入，详情和抽屉保持连续空间关系。
- 仅首页主专题和少量精选卡片启用克制磁吸：小幅位移、边缘高光、弹性回位，无粒子。
- 触屏、无悬停设备及 `prefers-reduced-motion` 自动关闭磁吸和大位移动效。
- 保留并改进当前歌词自动定位、主副歌词与字号层级，避免闪烁和突然聚焦。

## 组件边界

`PlayerApp` 保留业务编排职责，视图逐步拆为悬浮导航、编辑首页、搜索工作区、音乐库工作区、播放坞与沉浸详情。公共 API、stores、持久化格式和后端路由保持不变。

## 验收

桌面使用 1366×700，移动端使用 390×844。除单元测试与生产构建外，必须验证三页导航、播放、搜索、收藏、歌单、队列、歌词、账户与一起听，并以浏览器截图和概念图进行视觉对照。
